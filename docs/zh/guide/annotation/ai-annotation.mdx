# AI 自动标注

LabelU 支持接入 AI 模型服务，对图片数据进行自动标注。AI 自动标注生成的预标注结果可作为人工标注的起点，显著提升标注效率。

## 功能说明

- 目前仅支持**图片任务**的自动标注
- 支持的标注工具：拉框（rectTool）、多边形（polygonTool）
- AI 标注结果以预标注形式呈现，标注人员可在此基础上修改和完善
- 支持「只保留配置标签」选项——开启后仅返回与任务标签匹配的结果，关闭后模型自由检测所有目标

## 前置配置

AI 自动标注需要管理员进行以下配置后方可使用：

### 环境变量

在部署 LabelU 时设置以下环境变量：

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `AI_AUTO_LABEL_ENABLED` | 是否启用 AI 自动标注 | `false` |
| `AI_MODEL_ENDPOINT` | AI 模型服务地址（根路径） | 无 |
| `AI_MODEL_TIMEOUT_SECONDS` | 模型请求超时时间（秒） | `60` |
| `AI_PROVIDER` | AI 服务提供商标识 | 无 |
| `AI_MODEL_NAME` | 模型名称标识 | 无 |

示例配置：

```bash
export AI_AUTO_LABEL_ENABLED=true
export AI_MODEL_ENDPOINT=http://localhost:5000/
export AI_MODEL_TIMEOUT_SECONDS=60
export AI_PROVIDER=labelu
export AI_MODEL_NAME=florence-2-base
```

### 标注配置要求

任务的标注配置中至少需要一个支持的标注工具（拉框、多边形），且工具下需配置标签属性（attributes），AI 模型会根据这些标签生成对应的标注结果。

## 参考模型服务

LabelU 提供三套开源参考模型服务，均实现统一的 API 协议，开箱即用。

### 方案对比

| | Florence-2 | GroundingDINO + EfficientSAM | SAM 3 |
|---|---|---|---|
| 架构 | 单模型（检测+分割分步） | 两模型串联 | 单模型统一 |
| 开放词汇 | 支持 | 支持 | 支持（400万+概念） |
| 检测 (rectTool) | 一般 | 很好 | 很好 |
| 分割 (polygonTool) | 支持 | 精度高 | 精度最高 |
| 最低显存 | ~4GB | ~4GB | ~8GB |
| CPU 可用 | 可以（较慢） | 不推荐 | 不支持 |
| 模型大小 | ~500MB | ~900MB | ~1.7GB |
| Python 要求 | 3.8+ | 3.8+ | 3.12+ |
| CUDA 要求 | 可选 | 推荐 | 12.6+（必须） |

**推荐选择**：

- 有 4090/A100 等高端 GPU → **SAM 3**（质量最好，单模型最简单）
- 有中端 GPU（如 1660/2060） → **GroundingDINO + EfficientSAM**
- 只有 CPU 或显存紧张 → **Florence-2**

### 快速启动

#### Florence-2

```bash
cd model_server/florence2
pip install -r requirements.txt
python server.py --device cpu --port 5000
```

#### GroundingDINO + EfficientSAM

```bash
cd model_server/grounding_dino_sam
pip install -r requirements.txt
python server.py --device cuda --port 5000

# 仅检测（不加载 SAM，节省显存）
python server.py --device cuda --port 5000 --no-sam
```

#### SAM 3

```bash
# 需要 Python 3.12+, CUDA 12.6+, SM 80+ GPU (Ampere/Ada)
conda create -n sam3 python=3.12
conda activate sam3

cd model_server/sam3
pip install -r requirements.txt
python server.py --device cuda --port 5000
```

### Docker 部署

```bash
# Florence-2
cd model_server/florence2
docker build -t labelu-florence2 .
docker run -p 5000:5000 labelu-florence2

# GroundingDINO + EfficientSAM (GPU)
cd model_server/grounding_dino_sam
docker build -t labelu-dino-sam .
docker run --gpus all -p 5000:5000 labelu-dino-sam python server.py --device cuda

# SAM 3 (GPU, CUDA 12.6+)
cd model_server/sam3
docker build -t labelu-sam3 .
docker run --gpus all -p 5000:5000 labelu-sam3
```

2. 在右上角工具栏中，点击「AI 标注」按钮

### 「只保留配置标签」选项

点击 AI 标注按钮右侧的下拉箭头，可切换此选项：

- **开启**（默认）：仅保留与任务标注配置中标签名匹配的检测结果
- **关闭**：模型自由检测所有目标，标签名由模型自动生成

此设置会自动持久化，刷新页面后保留。

### 结果处理

- AI 标注结果会以**预标注**形式加载，标注人员可以直接在此基础上进行调整
- 再次点击「AI 标注」会覆盖之前的 AI 标注结果
- 手动标注的结果不受影响，AI 标注结果独立存储

:::tip
AI 标注结果仅作为参考，建议标注人员对结果进行人工审核和必要的修正，以确保标注质量。
:::

## 错误处理

使用 AI 标注时可能遇到以下提示：

| 错误提示 | 说明 | 解决方案 |
|---------|------|---------|
| AI annotation is not enabled | AI 标注未启用 | 联系管理员设置 `AI_AUTO_LABEL_ENABLED=true` |
| AI annotation only supports image tasks | 不支持的媒体类型 | AI 标注仅支持图片任务 |
| No annotation tools with labels configured | 未配置标注工具 | 在标注配置中添加支持的工具并配置标签 |
| AI model endpoint is not configured | 模型地址未配置 | 联系管理员设置 `AI_MODEL_ENDPOINT` |
| AI model service request failed | 模型服务请求失败 | 检查模型服务是否正常运行 |

## 模型服务 API 协议

如需接入自定义模型，只需实现以下 HTTP API 协议即可。

### POST /

**请求：**

```json
{
  "request_id": "uuid",
  "image_url": "https://presigned-or-public-url/image.jpg",
  "task": { "id": 10, "name": "demo-task" },
  "labels": [
    { "name": "car", "display_name": "汽车", "color": "#ff0000", "tool": "rectTool" },
    { "name": "person", "tool": "polygonTool" }
  ],
  "constraints": {
    "allowed_tools": ["rectTool", "polygonTool"],
    "max_results_per_label": 100,
    "filter_by_labels": true
  },
  "prompt": null
}
```

**constraints 字段说明：**

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `allowed_tools` | string[] | [] | 限制返回的工具类型，为空不限制 |
| `max_results_per_label` | int | 100 | 每个标签最多返回的结果数 |
| `filter_by_labels` | bool | false | 是否过滤掉不在 labels 列表中的检测结果 |

`filter_by_labels` 说明：
- `false`：模型检测到什么就返回什么，标签名作为 prompt 提示但不做过滤
- `true`：仅返回与配置标签名匹配的结果

**响应：**

```json
{
  "model": "microsoft/Florence-2-base",
  "latency_ms": 1840,
  "results": [
    {
      "toolName": "rectTool",
      "label": "car",
      "result": { "x": 120, "y": 80, "width": 260, "height": 140 },
      "score": 0.94
    },
    {
      "toolName": "polygonTool",
      "label": "person",
      "result": {
        "type": "line",
        "points": [
          { "x": 50, "y": 100 },
          { "x": 80, "y": 200 },
          { "x": 30, "y": 200 }
        ]
      },
      "score": 0.87
    }
  ],
  "warning_message": null
}
```

### GET /health

返回模型加载状态，用于健康检查。

### 各工具的 result 字段

| 工具 | result 格式 |
|------|------------|
| rectTool | `{ "x", "y", "width", "height" }` |
| polygonTool | `{ "type": "line", "points": [{"x", "y"}, ...] }` |

可选公共字段：`score`（置信度分数）
