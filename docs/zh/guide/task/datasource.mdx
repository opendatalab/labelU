# S3 数据源导入

除了本地上传文件外，LabelU 还支持从 S3 兼容的对象存储服务（如 AWS S3、MinIO、阿里云 OSS 等）中导入数据。

## 创建数据源

在导入数据之前，需要先创建一个数据源连接。

### 进入数据源管理

点击顶部导航栏的「数据源」菜单，进入数据源管理页面。

{/* TODO: 截图 - 数据源管理页面入口 */}

### 新建数据源

点击「新建数据源」按钮，填写以下配置：

| 配置项 | 说明 | 是否必填 |
|--------|------|---------|
| 名称 | 数据源名称，便于识别 | 是 |
| Endpoint | S3 服务地址，如 `https://s3.amazonaws.com` | 是 |
| Region | 区域，如 `us-east-1` | 否 |
| Bucket | 存储桶名称 | 是 |
| Prefix | 路径前缀，用于限定文件浏览范围 | 否 |
| Access Key ID | 访问密钥 ID | 是 |
| Secret Access Key | 访问密钥 | 是 |
| Path Style | 是否使用路径风格访问（MinIO 等自建存储通常需开启） | 否 |
| 使用 SSL | 是否使用 HTTPS 连接 | 否 |
| 预签名过期时间 | 预览链接有效期（秒），默认 3600 | 否 |

{/* TODO: 截图 - 新建数据源表单 */}

:::tip
- 对于 MinIO 等自建存储服务，需要开启「Path Style」选项
- Access Key 和 Secret Key 会加密存储，请放心填写
:::

## 从数据源导入文件

创建任务后，在「数据导入」步骤中可以选择从 S3 数据源导入。

### 打开 S3 导入弹窗

在数据导入页面，点击「S3 导入」按钮，打开导入弹窗。

{/* TODO: 截图 - 数据导入页面的 S3 导入按钮 */}

### 选择数据源

在弹窗中选择已创建的数据源，系统会自动列出该数据源下的文件。

{/* TODO: 截图 - S3 导入弹窗，显示数据源选择和文件列表 */}

### 浏览与导入

支持两种导入方式：

#### 方式一：导入所选

1. 在文件列表中勾选需要导入的文件
2. 点击底部「导入所选」按钮
3. 仅导入勾选的文件

#### 方式二：导入全部

1. 选择数据源后，无需勾选文件
2. 直接点击底部「导入全部」按钮
3. 系统会自动导入当前路径下所有匹配媒体格式的文件

{/* TODO: 截图 - S3 导入弹窗底部按钮区域（导入所选 + 导入全部） */}

:::info
- 导入全部功能会根据当前任务的媒体类型自动过滤文件扩展名（如图片任务只导入 jpg、png 等）
- 单次导入上限为 10000 个文件
- 已经导入过的同名文件不会重复导入
:::

### 导入结果

导入成功后，文件会出现在数据导入页面的上传列表中，显示为「上传成功」状态。

{/* TODO: 截图 - 导入成功后的文件列表 */}

## 文件预览

从 S3 导入的文件，在样本列表和标注页面中均可正常预览。系统会自动生成带有时效性的预签名 URL 用于文件访问。

:::note
预签名 URL 的有效期由数据源配置中的「预签名过期时间」决定，过期后刷新页面即可重新获取。
:::
