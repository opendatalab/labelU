# AI Auto-Annotation

LabelU supports integrating AI model services to automatically annotate image data. AI-generated pre-annotations can serve as a starting point for manual annotation, significantly improving annotation efficiency.

## Overview

- Currently only supports auto-annotation for **image tasks**
- Supported annotation tools: Rectangle (rectTool), Polygon (polygonTool)
- AI annotation results are presented as pre-annotations that annotators can modify and refine
- Supports "Only keep configured labels" option — when enabled, only results matching task labels are returned; when disabled, the model detects all objects freely
- Supports **single-image annotation** (per-image on annotation page) and **batch annotation** (one-click annotation for all unannotated samples on the sample list page)

## Architecture

![](/assets/arch.png)

Workflow:
1. User triggers AI annotation from the frontend (single or batch)
2. LabelU backend assembles the request (image URL, label list, constraints) and sends it to the model server via HTTP POST
3. Model server downloads the image and runs inference, returning detection/segmentation results
4. LabelU backend converts results into pre-annotation format and saves to database

:::warning Network Requirements
The model server must be able to access image URLs provided by LabelU. If LabelU and the model server are deployed on different machines, ensure the model server can reach the `MEDIA_HOST` address over the network.
:::

## Prerequisites

AI auto-annotation requires the following configuration by an administrator:

### Environment Variables

Set the following environment variables when deploying LabelU:

| Variable | Description | Default |
|----------|-------------|---------|
| `AI_AUTO_LABEL_ENABLED` | Enable AI auto-annotation | `false` |
| `AI_MODEL_ENDPOINT` | AI model service URL (root path) | None |
| `AI_MODEL_TIMEOUT_SECONDS` | Model request timeout (seconds) | `60` |
| `AI_PROVIDER` | AI service provider identifier | None |
| `AI_MODEL_NAME` | Model name identifier | None |

### Annotation Configuration Requirements

The task's annotation configuration must include at least one supported annotation tool (Rectangle or Polygon) with label attributes configured. The AI model generates annotation results based on these labels.

## Model Deployment

LabelU provides three open-source reference model server implementations, all following the same unified API protocol.

### System Requirements

| Resource | Florence-2 | GroundingDINO + EfficientSAM | SAM 3 |
|----------|-----------|------------------------------|-------|
| Python | 3.8+ | 3.8+ | **3.12+** |
| PyTorch | 2.1+ | 2.1+ | **2.7+** |
| CUDA | Optional | Recommended | **12.6+ (required)** |
| GPU Compute | Any | Any | **SM 80+ (Ampere/Ada)** |
| Min VRAM | ~4GB | ~4GB | ~8GB |
| Model Size | ~500MB | ~900MB | ~1.7GB |
| RAM | ≥8GB | ≥8GB | ≥16GB |
| Disk | ≥2GB (incl. model cache) | ≥3GB (incl. model cache) | ≥5GB (incl. model cache) |

### Comparison

| | Florence-2 | GroundingDINO + EfficientSAM | SAM 3 |
|---|---|---|---|
| Architecture | Single model (detect + segment) | Two models in pipeline | Single unified model |
| Open vocabulary | Yes | Yes | Yes (4M+ concepts) |
| Detection (rectTool) | Fair | Good | Good |
| Segmentation (polygonTool) | Supported | High precision | Highest precision |
| Min VRAM | ~4GB | ~4GB | ~8GB |
| CPU support | Yes (slow) | Not recommended | Not supported |
| Model size | ~500MB | ~900MB | ~1.7GB |
| Python | 3.8+ | 3.8+ | 3.12+ |
| CUDA | Optional | Recommended | 12.6+ (required) |

**Recommendations**:

- High-end GPU (4090/A100) → **SAM 3** (best quality, single model simplicity)
- Mid-range GPU (1660/2060) → **GroundingDINO + EfficientSAM**
- CPU only or low VRAM → **Florence-2**

### Installation & Startup

#### Florence-2

```bash
cd model_server/florence2
pip install -r requirements.txt
python server.py --device cpu --port 5000
```

#### GroundingDINO + EfficientSAM

```bash
cd model_server/grounding_dino_sam
pip install -r requirements.txt
python server.py --device cuda --port 5000

# Detection only (skip SAM to save VRAM)
python server.py --device cuda --port 5000 --no-sam
```

#### SAM 3

```bash
# Requires Python 3.12+, CUDA 12.6+, SM 80+ GPU (Ampere/Ada)
conda create -n sam3 python=3.12
conda activate sam3

cd model_server/sam3
pip install -r requirements.txt
python server.py --device cuda --port 5000
```

### Docker Deployment

```bash
# Florence-2
cd model_server/florence2
docker build -t labelu-florence2 .
docker run -p 5000:5000 labelu-florence2

# GroundingDINO + EfficientSAM (GPU)
cd model_server/grounding_dino_sam
docker build -t labelu-dino-sam .
docker run --gpus all -p 5000:5000 labelu-dino-sam python server.py --device cuda

# SAM 3 (GPU, CUDA 12.6+)
cd model_server/sam3
docker build -t labelu-sam3 .
docker run --gpus all -p 5000:5000 labelu-sam3
```

:::tip First-Time Startup
The first time you start a model server, it will automatically download model weights from HuggingFace. Download time depends on your network speed. Model weights are cached locally (usually at `~/.cache/huggingface/`) and won't need to be downloaded again on subsequent starts.

If your network is restricted, you can manually download the models in advance and set the `HF_HOME` environment variable to specify the cache directory.
:::

### Verify Deployment

After the model server starts successfully, verify with the health check endpoint:

```bash
curl http://localhost:5000/health
```

A `200` status code indicates the model is loaded and ready.

### Configure LabelU Connection

Once the model server is ready, configure the connection in LabelU's `.env` file or environment variables:

```bash
AI_AUTO_LABEL_ENABLED=true
AI_MODEL_ENDPOINT=http://localhost:5000/
AI_MODEL_TIMEOUT_SECONDS=300
AI_PROVIDER=labelu
AI_MODEL_NAME=florence-2-base     # or grounding-dino-tiny+efficient-sam / sam3
```

After restarting LabelU, the AI annotation button will appear on image task annotation pages and sample list pages.

## Usage

### Single-Image AI Annotation

1. Enter the image annotation page
2. Click the "AI Annotate" button in the top-right toolbar

![florence2](/assets/ai-annotation.jpeg)

3. The system sends the current image and task configuration to the AI model for inference
4. After the model returns results, annotations are automatically displayed on the image

![](/assets/ai-annotation-result.jpeg)
> 目标检测，使用 florence2 模型标注

![](/assets/ai-annotation-segment.jpeg)
> 目标分割，使用 florence2 模型标注

### Batch AI Annotation

For tasks with many samples, you can annotate all unannotated samples with one click:

1. On the **sample list page**, click the "Batch AI Annotate" button in the bottom toolbar
2. The system creates a background job that calls the AI model for each sample
3. A real-time progress bar shows success count, failure count, and total
4. After completion, the sample list refreshes automatically; annotators can enter the annotation page to review and correct AI-generated pre-annotations

:::info
Batch annotation only processes **unannotated** (NEW state) samples. Already annotated or skipped samples will not be overwritten.
:::

### "Only Keep Configured Labels" Option

Click the dropdown arrow on the right side of the AI Annotate button to toggle this option:

- **Enabled** (default): Only keeps detection results matching the label names in the task configuration
- **Disabled**: The model freely detects all objects; label names are auto-generated by the model

This setting is automatically persisted and retained after page refresh. Batch annotation also follows this setting.

### Result Handling

- AI annotation results are loaded as **pre-annotations** that annotators can adjust directly
- Clicking "AI Annotate" again will overwrite previous AI annotation results
- Manual annotations are not affected; AI annotation results are stored independently

:::tip
AI annotation results are provided as references. It is recommended that annotators review and correct the results to ensure annotation quality.
:::

## Error Handling

You may encounter the following messages when using AI annotation:

| Error Message | Description | Solution |
|---------------|-------------|----------|
| AI annotation is not enabled | AI annotation is disabled | Contact admin to set `AI_AUTO_LABEL_ENABLED=true` |
| AI annotation only supports image tasks | Unsupported media type | AI annotation only works with image tasks |
| No annotation tools with labels configured | No tools configured | Add supported tools with label attributes in annotation config |
| AI model endpoint is not configured | Model URL not set | Contact admin to set `AI_MODEL_ENDPOINT` |
| AI model service request failed | Model service error | Check if the model service is running properly |
| No unannotated samples | No samples to annotate | All samples are already annotated (batch mode) |

## Model Service API Protocol

To integrate a custom model, simply implement the following HTTP API protocol.

### POST /

**Request:**

```json
{
  "request_id": "uuid",
  "image_url": "https://presigned-or-public-url/image.jpg",
  "task": { "id": 10, "name": "demo-task" },
  "labels": [
    { "name": "car", "display_name": "Car", "color": "#ff0000", "tool": "rectTool" },
    { "name": "person", "tool": "polygonTool" }
  ],
  "constraints": {
    "allowed_tools": ["rectTool", "polygonTool"],
    "max_results_per_label": 100,
    "filter_by_labels": true
  },
  "prompt": null
}
```

**Constraints fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `allowed_tools` | string[] | [] | Restrict returned tool types; empty means no restriction |
| `max_results_per_label` | int | 100 | Maximum results per label |
| `filter_by_labels` | bool | false | Whether to filter out results not matching the labels list |

`filter_by_labels` explained:
- `false`: Model returns everything it detects; label names serve as prompt hints only
- `true`: Only returns results matching configured label names

**Response:**

```json
{
  "model": "microsoft/Florence-2-base",
  "latency_ms": 1840,
  "results": [
    {
      "toolName": "rectTool",
      "label": "car",
      "result": { "x": 120, "y": 80, "width": 260, "height": 140 },
      "score": 0.94
    },
    {
      "toolName": "polygonTool",
      "label": "person",
      "result": {
        "type": "line",
        "points": [
          { "x": 50, "y": 100 },
          { "x": 80, "y": 200 },
          { "x": 30, "y": 200 }
        ]
      },
      "score": 0.87
    }
  ],
  "warning_message": null
}
```

### GET /health

Returns model loading status for health checking.

### Result Fields by Tool

| Tool | Result format |
|------|--------------|
| rectTool | `{ "x", "y", "width", "height" }` |
| polygonTool | `{ "type": "line", "points": [{"x", "y"}, ...] }` |

Optional common fields: `score` (confidence score)
